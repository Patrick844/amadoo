### 3. Token refresh — automatic, transparent (implemented this conversation)

**Problem:** Access tokens expire. Every call returned `401 Unauthorized`. App crashed silently.

**Solution in `services/api.ts`:**

```
Request → 401?
  → call POST /auth/refresh (with stored refreshToken)
  → save new access_token + refresh_token to Zustand
  → retry original request automatically
  → user sees nothing
```

**Mutex pattern** — prevents duplicate refresh calls when multiple requests 401 simultaneously:
```ts
let refreshPromise: Promise<string> | null = null

if (!refreshPromise) {
  refreshPromise = doRefresh().finally(() => { refreshPromise = null })
}
const newToken = await refreshPromise  // all waiting calls share the same refresh
```

**If refresh itself fails** (refresh token expired — after weeks of inactivity) → `logout()` called, user sent to sign-in screen.

**Lazy import** used (`await import('@/stores/auth.store')`) inside `doRefresh()` to avoid circular dependency at module load time (api.ts ↔ auth.store.ts).

### 4. Swipe card — ghost card pattern (zero-lag transitions)

**Problem:** When a card was swiped, `dragX.value = 0` reset the card to center while React hadn't committed `advanceDeck()` yet — causing a visible snap-back flash.

**Solution — ghost exit card:**
1. When swipe threshold crossed → save swiped card's photo to `exitUri` state
2. Call `advanceDeck()` + reset `dragX = 0` immediately → new card appears at center with zero delay
3. A separate ghost card (`exitX` shared value) flies off screen independently over 550ms
4. Ghost card carries the **LIKE** (green) or **NOPE** (red) stamp so the user sees it during exit

**Key detail:** Ghost card always starts from ±60px off center (not from wherever the finger was). This ensures a consistent, visible 550ms animation regardless of swipe speed.

```ts
const direction = action === 'dislike' ? -1 : 1
exitX.value = direction * 60         // always start near center
exitX.value = withDelay(50,           // 50ms gives React time to render ghost first
  withTiming(direction * SCREEN_W * 1.6, { duration: 550 })
)
```

**Three cards pre-rendered** in a stack:
- `nextNextUser` at scale 0.88 (GPU-resident, two swipes ahead)
- `nextUser` at scale 0.93→1.0 (animates up as current card is dragged)
- `currentUser` (top, has gesture)

### 5. Match animation — full-screen redesign with Amadoo logo

**Old behavior:** A small card popped in with no animation — invisible.

**Why it was broken:** `entering` layout animations in Expo Go are skipped. Must use explicit `useSharedValue` + `useEffect`.

**New design (Tinder/Bumble style):**

```
t=0ms    Dark overlay fades in (400ms)
t=200ms  Amadoo logo drops in from above (spring)
t=450ms  "It's a Match!" rises from below (spring)
t=700ms  Two photos fly in from opposite sides (spring)
t=1600ms Buttons fade in — overlay stays on screen before dismissable
```

The full-screen overlay uses `StyleSheet.absoluteFillObject` with `backgroundColor: 'rgba(10,0,20,0.92)'`.
Logo: `require('../../Pages design/logo-hd-amadoo.png')` (white on transparent).
Photos: 120px circles with coral border, flying in from ±SCREEN_W/2.
Button: coral "Send a Message 💬", then "Keep Swiping" ghost text after 1.6s.

### 7. Swipe screen refresh — minimum loading duration

**Problem:** React batches `setLoading(true)` + `setLoading(false)` when the API is very fast → loading screen appeared for 1ms (one render frame).

**Fix:** `Promise.all` with a 4.5-second minimum wait:
```ts
const [cards] = await Promise.all([
  api.getDeck(token),
  new Promise<void>((r) => setTimeout(r, 4500)),
])
```
API and timer run in parallel. Loading screen shows for `max(api_duration, 4500ms)`.
Loading UI: `ActivityIndicator` + "Finding people near you..."

### 8. Push token fix

**Problem:** `getExpoPushTokenAsync({ projectId: 'amadoo' })` — `'amadoo'` is not a valid UUID. Backend returned 400.

**Fix in `services/notifications.ts`:**
```ts
// Before:
const tokenData = await Notifications.getExpoPushTokenAsync({ projectId: 'amadoo' })
// After:
const tokenData = await Notifications.getExpoPushTokenAsync()
```
Expo Go resolves the project ID automatically.

**Jade** (`jade@amadoo.com`) — Fashion Designer, AUB, face-verified, has a cat.

**Flow:**

4. API returns `matched: true` → match animation fires
5. Patrick can send a message or keep swiping

**Reset script** (use to re-test the match):
```bash
cd backend && source .venv/bin/activate && python -c "
import asyncio
from sqlalchemy import select, delete
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker
from config import settings
from models import User, Swipe, Match, Message

engine = create_async_engine(settings.DATABASE_URL, echo=False)
S = async_sessionmaker(engine, expire_on_commit=False)

async def reset():
    async with S() as db:
        jade_id = (await db.execute(select(User.id).where(User.email == 'jade@amadoo.com'))).scalar_one()
        patrick_id = (await db.execute(select(User.id).where(User.email == 'patrick-saade@hotmail.com'))).scalar_one()
        user_a, user_b = sorted([jade_id, patrick_id])
        match = (await db.execute(select(Match.id).where(Match.user_a_id == user_a, Match.user_b_id == user_b))).scalar_one_or_none()
        if match:
            await db.execute(delete(Message).where(Message.match_id == match))
            await db.execute(delete(Match).where(Match.id == match))
        await db.execute(delete(Swipe).where(
            ((Swipe.swiper_id == jade_id) & (Swipe.swiped_id == patrick_id)) |
            ((Swipe.swiper_id == patrick_id) & (Swipe.swiped_id == jade_id))
        ))
        db.add(Swipe(swiper_id=jade_id, swiped_id=patrick_id, action='like'))
        await db.commit()
        print('Done')

asyncio.run(reset())
"
```
